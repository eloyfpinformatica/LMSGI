5.2. Conceptos Fundamentales de APIs ​

API utilizada: JSONPlaceholder — https://jsonplaceholder.typicode.com
Autenticación: Ninguna
Objetivo: Ver en la práctica qué es un endpoint, cómo se usan los verbos HTTP y qué significan los códigos de estado.

¿Qué vamos a hacer? ​

Cinco botones, cinco operaciones sobre el mismo recurso (/posts). Cada vez que pulsas uno, la app hace una petición real a JSONPlaceholder y muestra en pantalla:

  • La URL exacta que se ha pedido
  • El verbo HTTP utilizado
  • El código de estado de la respuesta
  • El JSON que ha devuelto el servidor

Nota: JSONPlaceholder es una API de pruebas. Las operaciones POST, PUT, PATCH y DELETE simulan el comportamiento correcto pero no modifican datos reales en ningún servidor. Es perfecta para aprender sin riesgos.

    <script>
        const BASE_URL = 'https://jsonplaceholder.typicode.com/posts';

        // Configuración de cada verbo: URL, opciones fetch y descripción
        const configuraciones = {
            GET: {
                url: `${BASE_URL}/1`,
                opciones: { method: 'GET' },
                descripcion: 'Leemos el post con ID 1. No modificamos nada en el servidor.'
            },
            POST: {
                url: BASE_URL,
                opciones: {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify({
                        title: 'Mi nuevo post',
                        body:  'Contenido del post',
                        userId: 1
                    })
                },
                descripcion: 'Creamos un nuevo post. El servidor responde con el recurso creado y su nuevo ID.'
            },
            PUT: {
                url: `${BASE_URL}/1`,
                opciones: {
                    method: 'PUT',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify({
                        id:     1,
                        title:  'Título completamente nuevo',
                        body:   'Cuerpo completamente nuevo',
                        userId: 1
                    })
                },
                descripcion: 'Reemplazamos el post 1 por completo. Hay que enviar todos los campos.'
            },
            PATCH: {
                url: `${BASE_URL}/1`,
                opciones: {
                    method: 'PATCH',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify({
                        title: 'Solo cambio el título'
                    })
                },
                descripcion: 'Modificamos solo el título del post 1. El resto de campos no cambia.'
            },
            DELETE: {
                url: `${BASE_URL}/1`,
                opciones: { method: 'DELETE' },
                descripcion: 'Eliminamos el post 1. El servidor devuelve un objeto vacío y código 200.'
            }
        };

        // Colores asociados a cada verbo (para la etiqueta)
        const coloresVerbo = {
            GET:    '#2ecc71',
            POST:   '#3498db',
            PUT:    '#f39c12',
            PATCH:  '#9b59b6',
            DELETE: '#e74c3c'
        };

        async function hacerPeticion(verbo) {
            const { url, opciones, descripcion } = configuraciones[verbo];

            try {
                const respuesta = await fetch(url, opciones);
                const datos     = await respuesta.json();
                const codigo    = respuesta.status;

                // Rellenamos la sección de resultado
                const verboEl      = document.getElementById('verboEl');
                const urlEl        = document.getElementById('urlEl');
                const codigoEl     = document.getElementById('codigoEl');
                const descripcionEl = document.getElementById('descripcionEl');
                const jsonEl       = document.getElementById('jsonEl');

                verboEl.textContent            = verbo;
                verboEl.style.background       = coloresVerbo[verbo];
                urlEl.textContent              = `${opciones.method}  ${url}`;
                codigoEl.textContent           = `${codigo} ${respuesta.statusText}`;
                codigoEl.className             = `codigo ${codigo < 400 ? 'ok' : 'error'}`;
                descripcionEl.textContent      = descripcion;
                jsonEl.textContent             = JSON.stringify(datos, null, 2);

                document.getElementById('resultado').style.display = 'block';

            } catch (error) {
                console.error('Error en la petición:', error);
            }
        }
    </script>

Qué observar en cada petición ​

VerboCódigo esperadoPor qué
GET200 OKEl recurso existe y se devuelve correctamente
POST201 CreatedSe ha creado un nuevo recurso
PUT200 OKEl recurso se ha reemplazado
PATCH200 OKEl recurso se ha modificado parcialmente
DELETE200 OKEl recurso se ha eliminado (devuelve {})

Para practicar en clase ​

  1. Abre las DevTools → Network y pulsa cada botón. Localiza en cada petición el verbo, la URL y el código de estado.
  2. Cambia el id de la URL en GET a uno que no exista, por ejemplo /posts/99999. ¿Qué código devuelve?
  3. En el POST, elimina el campo title del body. ¿Cambia algo en la respuesta?
  4. Fíjate en la diferencia entre el body que envías en PUT y en PATCH. ¿Por qué PUT necesita todos los campos y PATCH no?

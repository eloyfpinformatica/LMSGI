5.4. Consumo de APIs con Fetch ​

API utilizada: PokéAPI — https://pokeapi.co
Autenticación: Ninguna
Objetivo: Ver todos los aspectos de fetch en una aplicación real: parámetros, headers, errores, rate limiting y transformación de datos.

¿Qué vamos a hacer? ​

Un buscador de Pokémon completo que demuestra:

  • Peticiones GET con diferentes URLs (por nombre, por ID)
  • Headers personalizados (opcional pero educativo)
  • Manejo de errores (Pokémon inexistente, problemas de red)
  • Transformación de datos (la API devuelve datos complejos que simplificamos)
  • Rate limiting (PokeAPI tiene límite de 100 peticiones/minuto por IP)

    <script>

        // -------------------------------------------------------
        // FUNCIÓN PRINCIPAL DE BÚSQUEDA
        // -------------------------------------------------------

        async function buscar(query) {
            const url = `https://pokeapi.co/api/v2/pokemon/${query.toLowerCase()}`;

            // 1. Mostrar estado de carga
            mostrarCargando();

            // Variables para medir el tiempo de respuesta
            const inicio = performance.now();

            try {
                // 2. PETICIÓN CON HEADERS PERSONALIZADOS (opcional pero educativo)
                const respuesta = await fetch(url, {
                    method: 'GET',  // Por defecto, pero lo especificamos para claridad
                    headers: {
                        'Accept': 'application/json',
                        'User-Agent': 'Pokedex-Educativa/1.0'
                        // PokéAPI no requiere estos headers, pero es buena práctica incluirlos
                    }
                });

                const tiempoRespuesta = (performance.now() - inicio).toFixed(0);

                // 3. MANEJO DE ERRORES HTTP
                if (!respuesta.ok) {
                    if (respuesta.status === 404) {
                        throw new Error('Pokémon no encontrado. Verifica el nombre o número.');
                    }
                    throw new Error(`Error HTTP ${respuesta.status}: ${respuesta.statusText}`);
                }

                // 4. Convertir respuesta a JSON
                const pokemon = await respuesta.json();

                // 5. MOSTRAR INFO DE LA PETICIÓN (educativo)
                mostrarInfoAPI(url, respuesta.status, tiempoRespuesta);

                // 6. TRANSFORMAR DATOS
                const pokemonSimplificado = transformarDatos(pokemon);

                // 7. RENDERIZAR
                renderizarPokemon(pokemonSimplificado);
                mostrarTarjeta();

            } catch (error) {
                // ERRORES: red, 404, JSON inválido, etc.
                mostrarError(error.message);
            }
        }


        // -------------------------------------------------------
        // TRANSFORMACIÓN DE DATOS
        // La API devuelve mucha información. Extraemos solo lo necesario.
        // -------------------------------------------------------

        function transformarDatos(pokemon) {
            return {
                id: pokemon.id,
                nombre: pokemon.name,
                sprite: pokemon.sprites.front_default,
                tipos: pokemon.types.map(t => t.type.name),
                stats: pokemon.stats.map(s => ({
                    nombre: s.stat.name,
                    valor: s.base_stat
                })),
                habilidades: pokemon.abilities.map(a => a.ability.name)
            };
        }


        // -------------------------------------------------------
        // RENDERIZADO
        // -------------------------------------------------------

        function renderizarPokemon(pokemon) {
            // Cabecera
            document.getElementById('sprite').src = pokemon.sprite;
            document.getElementById('pokemonName').textContent = pokemon.nombre;
            document.getElementById('pokemonNumber').textContent = `#${pokemon.id.toString().padStart(3, '0')}`;

            // Tipos
            const typesContainer = document.getElementById('types');
            typesContainer.innerHTML = pokemon.tipos
                .map(tipo => `<span class="type-badge type-${tipo}">${tipo}</span>`)
                .join('');

            // Stats
            const statsContainer = document.getElementById('statsContainer');
            statsContainer.innerHTML = pokemon.stats
                .map(stat => {
                    const porcentaje = (stat.valor / 255) * 100; // 255 es el máximo teórico
                    return `
                        <div class="stat-row">
                            <span class="stat-name">${stat.nombre}</span>
                            <div class="stat-bar-container">
                                <div class="stat-bar" style="width: ${porcentaje}%"></div>
                            </div>
                            <span class="stat-value">${stat.valor}</span>
                        </div>
                    `;
                })
                .join('');

            // Habilidades
            const abilitiesContainer = document.getElementById('abilitiesContainer');
            abilitiesContainer.innerHTML = pokemon.habilidades
                .map(hab => `<span class="ability-badge">${hab}</span>`)
                .join('');
        }


        // -------------------------------------------------------
        // GESTIÓN DE ESTADOS DE UI
        // -------------------------------------------------------

        function mostrarCargando() {
            document.getElementById('loading').style.display = 'block';
            document.getElementById('error').style.display = 'none';
            document.getElementById('pokemon-card').style.display = 'none';
            document.getElementById('apiInfo').classList.remove('visible');
            document.getElementById('searchBtn').disabled = true;
        }

        function mostrarTarjeta() {
            document.getElementById('loading').style.display = 'none';
            document.getElementById('pokemon-card').style.display = 'block';
            document.getElementById('searchBtn').disabled = false;
        }

        function mostrarError(mensaje) {
            document.getElementById('loading').style.display = 'none';
            document.getElementById('error').style.display = 'block';
            document.getElementById('errorMsg').textContent = `❌ ${mensaje}`;
            document.getElementById('searchBtn').disabled = false;
        }

        function mostrarInfoAPI(url, status, tiempo) {
            document.getElementById('apiUrl').textContent = url.split('/').pop();
            document.getElementById('apiStatus').textContent = `${status} OK`;
            document.getElementById('apiTime').textContent = `${tiempo} ms`;
            document.getElementById('apiInfo').classList.add('visible');
        }


        // -------------------------------------------------------
        // EVENTOS
        // -------------------------------------------------------

        document.getElementById('searchBtn').addEventListener('click', () => {
            const query = document.getElementById('searchInput').value.trim();
            if (query) buscar(query);
        });

        document.getElementById('searchInput').addEventListener('keypress', e => {
            if (e.key === 'Enter') {
                const query = document.getElementById('searchInput').value.trim();
                if (query) buscar(query);
            }
        });


        // -------------------------------------------------------
        // CARGA INICIAL
        // -------------------------------------------------------

        buscar('pikachu');

    </script>

</body>

Conceptos clave que demuestra este ejemplo ​

1. Headers personalizados ​

javascript
headers: {
    'Accept': 'application/json',
    'User-Agent': 'Pokedex-Educativa/1.0'
}

PokéAPI no requiere estos headers, pero es buena práctica enviar Accept para indicar qué formato esperas, y User-Agent para identificar tu aplicación.

2. Manejo de errores HTTP ​

javascript
if (!respuesta.ok) {
    if (respuesta.status === 404) {
        throw new Error('Pokémon no encontrado');
    }
    throw new Error(`Error HTTP ${respuesta.status}`);
}

fetch() no rechaza la promesa con códigos 4xx o 5xx. Tenemos que comprobarlo manualmente.

3. Transformación de datos ​

La API devuelve un JSON enorme. La función transformarDatos() extrae solo lo que necesitamos y lo simplifica:

javascript
// De esto (API):
{ types: [ { type: { name: "electric" } } ] }

// A esto (nuestra app):
{ tipos: ["electric"] }

4. Rate limiting ​

PokéAPI permite 100 peticiones por minuto por IP. Si haces búsquedas muy rápidas, verás un error 429. En el código real de producción, implementarías:

  • Caché de resultados previos
  • Debounce en el input
  • Cola de peticiones

5. Medición de rendimiento ​

javascript
const inicio = performance.now();
// ... petición ...
const tiempo = performance.now() - inicio;

Esto demuestra cómo medir el tiempo de respuesta de la API.

Ejercicios para clase ​

  1. Provoca un error 404 buscando pokemon-inventado. Observa cómo el código detecta el 404 y muestra un mensaje personalizado.

  2. Abre DevTools → Network y busca varios Pokémon. Observa:

    • Los headers de la petición (Request Headers)
    • Los headers de la respuesta (Response Headers)
    • El tamaño del JSON recibido

  3. Busca muy rápido (5-10 búsquedas en pocos segundos). ¿Recibes algún error de rate limit?

  4. Modifica el código: añade un botón para ver la "forma shiny" del Pokémon. El sprite está en pokemon.sprites.front_shiny.

Comparativa con APIs que requieren autenticación ​

PokéAPI no requiere API key. Otras APIs (como TMDB, OpenWeatherMap) sí. La diferencia en el código sería:

javascript
// Con API key en la URL
const url = `https://api.ejemplo.com/data?apikey=${MI_API_KEY}`;

// O con API key en headers
headers: {
    'Authorization': `Bearer ${MI_TOKEN}`
}

El resto del código (fetch, async/await, manejo de errores) es idéntico.

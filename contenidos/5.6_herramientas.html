5.6. Herramientas de Desarrollo ​

Objetivo: Dominar las herramientas profesionales para trabajar con APIs: Postman y las DevTools del navegador.

🛠️ Parte 1: Postman / Insomnia ​

¿Qué es Postman? ​

Postman es una aplicación que permite probar APIs sin escribir código. Es la herramienta estándar en la industria para:

  • Explorar APIs nuevas antes de integrarlas
  • Probar endpoints con diferentes parámetros
  • Depurar respuestas inesperadas
  • Documentar APIs
  • Compartir colecciones de peticiones con el equipo

Alternativa: Insomnia — funciona de forma muy similar, interfaz más minimalista.

Práctica guiada 1: Primera petición en Postman ​

API a usar: JSONPlaceholder — https://jsonplaceholder.typicode.com

Paso 1: Crear una petición GET ​

┌─────────────────────────────────────────────────────────────┐
│ Postman                                          [ - ] [ × ] │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  [GET ▼]  https://jsonplaceholder.typicode.com/users/1  [Send]│
│                                                               │
│  Params | Authorization | Headers | Body | Pre-request | Tests│
│                                                               │
└─────────────────────────────────────────────────────────────┘

Instrucciones:

  1. Abre Postman y haz clic en "New Request" o presiona Ctrl+N
  2. Método: Deja GET seleccionado (es el predeterminado)
  3. URL: Pega https://jsonplaceholder.typicode.com/users/1
  4. Haz clic en Send

Paso 2: Examinar la respuesta ​

┌─────────────────────────────────────────────────────────────┐
│ Response                                                      │
├─────────────────────────────────────────────────────────────┤
│  Body | Cookies | Headers | Test Results                     │
│                                                               │
│  Status: 200 OK    Time: 247 ms    Size: 508 B               │
│                                                               │
│  Pretty | Raw | Preview                                       │
│                                                               │
│  {                                                            │
│    "id": 1,                                                   │
│    "name": "Leanne Graham",                                  │
│    "username": "Bret",                                        │
│    "email": "Sincere@april.biz",                             │
│    "address": {                                               │
│      "street": "Kulas Light",                                │
│      "city": "Gwenborough",                                  │
│      ...                                                      │
│    }                                                          │
│  }                                                            │
└─────────────────────────────────────────────────────────────┘

Observa:

  • Status: 200 OK → La petición fue exitosa
  • Time: 247 ms → El servidor tardó 247 milisegundos en responder
  • Size: 508 B → El JSON pesa 508 bytes
  • El JSON está formateado automáticamente (Pretty mode)

Paso 3: Añadir parámetros de consulta (Query Params) ​

Vamos a buscar posts del usuario 1.

URL con parámetros: https://jsonplaceholder.typicode.com/posts?userId=1

En Postman hay dos formas de hacerlo:

Forma A — Escribir la URL completa:

https://jsonplaceholder.typicode.com/posts?userId=1

Forma B — Usar la pestaña Params (recomendado):

┌─────────────────────────────────────────────────────────────┐
│  [GET]  https://jsonplaceholder.typicode.com/posts          │
│                                                               │
│  Params | Authorization | Headers | Body                     │
│  ─────────────────────────────────────────────────────────   │
│                                                               │
│  Query Params:                                                │
│  ☑ userId     1         [Description...]                     │
│  ☐ _limit     10        [Description...]                     │
│                                                               │
│  [+ Add new parameter]                                        │
└─────────────────────────────────────────────────────────────┘

Ventaja de Params: Puedes activar/desactivar parámetros con el checkbox sin editar la URL.

Paso 4: Ver los Headers de la petición ​

Haz clic en la pestaña Headers para ver qué headers se están enviando:

┌─────────────────────────────────────────────────────────────┐
│  Headers (7)                                                  │
│  ─────────────────────────────────────────────────────────   │
│                                                               │
│  Request Headers:                                             │
│  ☑ User-Agent        PostmanRuntime/7.32.1                   │
│  ☑ Accept            */*                                      │
│  ☑ Cache-Control     no-cache                                │
│  ☑ Host              jsonplaceholder.typicode.com            │
│  ☑ Accept-Encoding   gzip, deflate, br                       │
│                                                               │
│  [+ Add new header]                                           │

Postman añade headers automáticamente, pero puedes añadir los tuyos propios.

Práctica guiada 2: Petición POST ​

Ahora vamos a crear un nuevo post usando el método POST.

Paso 1: Cambiar el método a POST ​

┌─────────────────────────────────────────────────────────────┐
│  [POST ▼] https://jsonplaceholder.typicode.com/posts   [Send]│
└─────────────────────────────────────────────────────────────┘

Paso 2: Añadir el Body (cuerpo de la petición) ​

Haz clic en la pestaña Body y selecciona raw → JSON:

┌─────────────────────────────────────────────────────────────┐
│  Body                                                         │
│  ─────────────────────────────────────────────────────────   │
│                                                               │
│  ○ none  ○ form-data  ○ x-www-form-urlencoded                │
│  ● raw   ○ binary     ○ GraphQL                              │
│                                                               │
│  [Text ▼]  →  [JSON ▼]                                       │
│                                                               │
│  {                                                            │
│    "title": "Mi primer post desde Postman",                  │
│    "body": "Este es el contenido del post",                  │
│    "userId": 1                                                │
│  }                                                            │
│                                                               │
└─────────────────────────────────────────────────────────────┘

Paso 3: Enviar y ver la respuesta ​

Haz clic en Send. El servidor responde con el post creado:

Status: 201 Created    Time: 312 ms

{
  "id": 101,
  "title": "Mi primer post desde Postman",
  "body": "Este es el contenido del post",
  "userId": 1
}

Observa:

  • El código es 201 Created (no 200) porque se creó un recurso nuevo
  • El servidor le asignó el id: 101

Práctica guiada 3: Guardar en una Colección ​

Las colecciones te permiten organizar y reutilizar peticiones.

Paso 1: Crear una colección ​

Sidebar de Postman:

Collections
  └─ [+] New Collection
      
      Nombre: "JSONPlaceholder API"
      Descripción: "Práctica de clase"

Paso 2: Guardar la petición ​

Después de hacer una petición, haz clic en Save (arriba a la derecha):

┌─────────────────────────────────────────────────────────────┐
│  Save Request                                                 │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  Request name:                                                │
│  [ Obtener usuario por ID                              ]     │
│                                                               │
│  Save to collection:                                          │
│  [▼] JSONPlaceholder API                                     │
│                                                               │
│                               [Cancel]  [Save to Collection] │
└─────────────────────────────────────────────────────────────┘

Ahora la petición está guardada y puedes ejecutarla cuando quieras sin reescribir nada.

Práctica guiada 4: Variables de entorno ​

Las variables evitan repetir URLs base y API keys.

Paso 1: Crear un entorno ​

Environments (icono de engranaje arriba a la derecha)
  → [+] Add
  
  Environment name: "Desarrollo"
  
  Variables:
    BASE_URL      https://jsonplaceholder.typicode.com
    API_KEY       (vacío, es solo un ejemplo)

Paso 2: Usar la variable en las peticiones ​

Cambia la URL de:

https://jsonplaceholder.typicode.com/users/1

A:

{{BASE_URL}}/users/1

Postman sustituye automáticamente por el valor configurado.

Ventaja: Si la API cambia de dominio, solo cambias la variable en un sitio.

Práctica guiada 5: Probar una API con autenticación ​

Vamos a probar OpenWeatherMap que requiere API key.

URL: https://api.openweathermap.org/data/2.5/weather

Paso 2: Configurar la petición ​

[GET] https://api.openweathermap.org/data/2.5/weather

Params:
  ☑ q       Madrid
  ☑ appid   TU_API_KEY_AQUI
  ☑ units   metric
  ☑ lang    es

Paso 3: Enviar ​

Status: 200 OK

{
  "weather": [
    {
      "main": "Clear",

    }
  ],
  "main": {
    "temp": 18.5,
    "feels_like": 17.8,
    "humidity": 65

🌐 Parte 2: DevTools del navegador ​

Las DevTools (Herramientas de Desarrollo) del navegador son esenciales para depurar aplicaciones web.

Cómo abrir las DevTools ​

  • Windows/Linux: F12 o Ctrl + Shift + I
  • Mac: Cmd + Option + I
  • Cualquier sistema: Click derecho → "Inspeccionar"

Práctica guiada 6: Pestaña Network (Red) ​

La pestaña Network muestra todas las peticiones HTTP que hace tu página.

Paso 1: Abrir un ejemplo ​

Copia este código en un archivo test.html:

html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Test Network</title>
</head>
<body>
    <h1>Prueba de Network DevTools</h1>
    <button id="btn">Cargar usuarios</button>
    <div id="resultado"></div>

    <script>
        document.getElementById('btn').addEventListener('click', async () => {
            const response = await fetch('https://jsonplaceholder.typicode.com/users');
            const usuarios = await response.json();
            document.getElementById('resultado').textContent = 
                `Cargados ${usuarios.length} usuarios`;
        });
    </script>
</body>
</html>

Paso 2: Inspeccionar la petición ​

  1. Abre test.html en el navegador
  2. Abre DevTools (F12)
  3. Ve a la pestaña Network / Red
  4. Haz clic en el botón "Cargar usuarios"

Verás algo así:

Práctica guiada 7: Filtrar peticiones ​

Cuando una página hace muchas peticiones, los filtros son esenciales.

Filtros disponibles:

[All]     → Todas las peticiones
[XHR]     → Solo peticiones fetch/AJAX (las que nos interesan para APIs)
[JS]      → Solo archivos JavaScript
[CSS]     → Solo hojas de estilo
[Img]     → Solo imágenes
[Doc]     → Solo documentos HTML

Buscar por texto:

Escribe en el cuadro de búsqueda:

[🔍 users]  → Filtra por nombre de archivo

Filtrar por código de estado:

Escribe en el cuadro:

status-code:404  → Solo peticiones con error 404
status-code:200  → Solo peticiones exitosas

Práctica guiada 8: Copiar como código ​

DevTools puede generar código automáticamente a partir de una petición.

Paso 1: Hacer click derecho en una petición ​

users  [Click derecho]
  └─ Copy
      ├─ Copy as cURL
      ├─ Copy as fetch
      └─ Copy as PowerShell

Paso 2: Seleccionar "Copy as fetch" ​

Te copia esto al portapapeles:

javascript
fetch("https://jsonplaceholder.typicode.com/users", {
  "headers": {
    "accept": "*/*",
    "accept-language": "es-ES,es;q=0.9",
    "cache-control": "no-cache",
  },
  "referrer": "http://localhost:5500/test.html",
  "referrerPolicy": "strict-origin-when-cross-origin",
  "body": null,
  "method": "GET",
  "mode": "cors",
  "credentials": "omit"
});

Uso: Puedes pegar esto directamente en tu código y funcionará.

🎯 Parte 3: Console (Consola) ​

La consola es donde JavaScript muestra mensajes y donde puedes ejecutar código en vivo.

Práctica guiada 10: Ejecutar fetch() directamente en la consola ​

Abre DevTools → pestaña Console.

Paso 1: Petición GET simple ​

Escribe y presiona Enter:

javascript
fetch('https://jsonplaceholder.typicode.com/users/1')
  .then(r => r.json())
  .then(console.log);

Output:

javascript
{id: 1, name: "Leanne Graham", username: "Bret", ...}

Paso 2: Con async/await (más limpio) ​

Escribe:

javascript
const response = await fetch('https://jsonplaceholder.typicode.com/users/1');
const usuario = await response.json();
console.log(usuario);

Ventaja: Puedes explorar la API sin escribir un archivo HTML.

Cuando la API devuelve muchos objetos, console.table() los muestra en formato tabla.

Práctica guiada 12: console.time() para medir rendimiento ​

javascript
console.time('carga-usuarios');

const response = await fetch('https://jsonplaceholder.typicode.com/users');
const usuarios = await response.json();

console.timeEnd('carga-usuarios');

Output:

carga-usuarios: 247.35ms

Práctica guiada 13: Debuggear respuestas inesperadas ​

Cuando una API devuelve datos que no esperabas:

javascript
const response = await fetch('https://jsonplaceholder.typicode.com/posts/1');
const post = await response.json();

// Ver todas las propiedades del objeto
console.dir(post);

// Ver qué tipo de dato es cada campo
console.log(typeof post.id);        // "number"
console.log(typeof post.title);     // "string"
console.log(Array.isArray(post));   // false

📝 Resumen: Flujo de trabajo recomendado ​

Al explorar una API nueva: ​

  1. Lee la documentación (si existe)
  2. Prueba en Postman primero:
    • Haz peticiones GET para entender la estructura de datos
    • Prueba POST/PUT/DELETE si la API lo permite
    • Guarda las peticiones en una colección
  3. Copia el código (Copy as fetch)
  4. Pégalo en tu aplicación y adáptalo
  5. Usa DevTools Network para depurar si algo falla

Al depurar un problema: ​

  1. Abre Network en DevTools
  2. Reproduce el error
  3. Busca la petición que falló
  4. Examina:
    • ¿El código de estado es correcto?
    • ¿Los headers son los esperados?
    • ¿El body es válido?
    • ¿El servidor devolvió JSON válido?
  5. Prueba la misma petición en Postman para aislar el problema

Ejercicio 3: Console avanzado ​

Ejecuta esto en la consola:

javascript
const response = await fetch('https://jsonplaceholder.typicode.com/users');
const usuarios = await response.json();

// ¿Cuántos usuarios hay?
console.log('Total:', usuarios.length);

// Ver solo nombres en tabla
console.table(usuarios.map(u => ({ nombre: u.name, email: u.email })));

// Encontrar al usuario con id 5
const usuario5 = usuarios.find(u => u.id === 5);
console.log('Usuario 5:', usuario5.name);

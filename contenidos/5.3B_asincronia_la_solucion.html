5.3B. Programación Asíncrona — La Solución ​

API utilizada: REST Countries — https://restcountries.com
Objetivo: Reescribir el ejemplo del punto 5.1 usando async/await y try/catch, y entender qué está pasando en cada paso.

¿Qué vamos a hacer? ​

Recuperamos el ejemplo del 5.1 — pedir información de un país — pero ahora:

  • El alumno puede buscar cualquier país, no solo España
  • Usamos async/await en lugar de .then()
  • Añadimos try/catch para manejar errores reales (país no encontrado, sin conexión...)
  • Provocamos errores a propósito para ver qué pasa

La idea es que el alumno vea el mismo problema que en 5.1 resuelto ahora de forma correcta, y que entienda cada línea del código.

        // -------------------------------------------------------
        // Esta es la función principal. Fíjate en la estructura:
        //
        //   async function = función que contiene código asíncrono
        //   await          = "espera aquí hasta que esto termine,
        //                     pero sin bloquear la página"
        //   try / catch    = si algo falla, ve al bloque catch
        // -------------------------------------------------------

        async function buscarPais(nombre) {
            const url = `https://restcountries.com/v3.1/name/${nombre}?fullText=true`;

            // 1. Preparamos la UI antes de la petición
            mostrarCargando();

            try {
                // 2. Hacemos la petición y ESPERAMOS la respuesta
                //    JavaScript no se bloquea — solo esta función espera
                const respuesta = await fetch(url);

                // 3. fetch() no lanza error si el servidor responde con 404.
                //    Tenemos que comprobarlo nosotros manualmente.
                if (!respuesta.ok) {
                    throw new Error(`País no encontrado (código ${respuesta.status})`);
                }

                // 4. Convertimos la respuesta a JSON y ESPERAMOS
                const datos = await respuesta.json();
                const pais  = datos[0];

                // 5. Extraemos los datos que necesitamos
                const monedas  = Object.values(pais.currencies || {});
                const moneda   = monedas.length > 0
                    ? `${monedas[0].name} (${monedas[0].symbol})`
                    : 'Sin datos';

                const idiomas  = Object.values(pais.languages || {}).join(', ') || 'Sin datos';

                // 6. Pintamos los datos en el DOM
                document.getElementById('bandera').src          = pais.flags.svg;
                document.getElementById('nombre-pais').textContent   = pais.name.common;
                document.getElementById('nombre-oficial').textContent = pais.name.official;
                document.getElementById('capital').textContent   = pais.capital?.[0] ?? 'Sin datos';
                document.getElementById('continente').textContent = pais.region;
                document.getElementById('poblacion').textContent  = pais.population.toLocaleString('es-ES') + ' hab.';
                document.getElementById('idiomas').textContent    = idiomas;
                document.getElementById('moneda').textContent     = moneda;

                mostrarTarjeta();

            } catch (error) {
                // Si fetch falla (sin red) o lanzamos un error manualmente,
                // llegamos aquí. El mensaje es el que pasamos a new Error().
                mostrarError(error.message);
            }
        }


        // --- Funciones de UI ---

        function mostrarCargando() {
            document.getElementById('cargando').style.display = 'block';
            document.getElementById('error').style.display   = 'none';
            document.getElementById('tarjeta').style.display = 'none';
            document.getElementById('btnBuscar').disabled    = true;
        }

        function mostrarTarjeta() {
            document.getElementById('cargando').style.display = 'none';
            document.getElementById('tarjeta').style.display  = 'block';
            document.getElementById('btnBuscar').disabled     = false;
        }

        function mostrarError(mensaje) {
            document.getElementById('cargando').style.display = 'none';
            document.getElementById('error').style.display    = 'block';
            document.getElementById('error').textContent      = `❌ ${mensaje}`;
            document.getElementById('btnBuscar').disabled     = false;
        }


        // --- Eventos ---

        document.getElementById('btnBuscar').addEventListener('click', () => {
            const pais = document.getElementById('inputPais').value.trim();
            if (pais) buscarPais(pais);
        });

        document.getElementById('inputPais').addEventListener('keypress', e => {
            if (e.key === 'Enter') {
                const pais = document.getElementById('inputPais').value.trim();
                if (pais) buscarPais(pais);
            }
        });


        // --- Carga inicial ---
        buscarPais('Spain');

    </script>

Comparativa: el mismo código en .then() y en async/await ​

El ejemplo del 5.1 hacía esto con .then():

javascript
fetch(URL)
    .then(respuesta => respuesta.json())
    .then(datos => {
        const pais = datos[0];
        // pintar datos...
    })
    .catch(error => {
        // manejar error...
    });

Ahora lo hacemos así con async/await:

javascript
async function buscarPais(nombre) {
    try {
        const respuesta = await fetch(url);
        const datos     = await respuesta.json();
        const pais      = datos[0];
        // pintar datos...
    } catch (error) {
        // manejar error...
    }
}

Son equivalentes. La segunda versión se lee de arriba a abajo, como código síncrono normal.

Para provocar errores a propósito ​

PruebaQué ocurre
Escribe paisinventadoLa API devuelve 404 → el throw new Error del if (!respuesta.ok) activa el catch
Desactiva el WiFi y buscafetch() falla → el catch captura el error de red
Borra el try/catch y repiteEl error no se captura y la app se rompe silenciosamente

La tercera prueba es especialmente útil en clase: demuestra por qué el try/catch no es opcional cuando trabajas con APIs.

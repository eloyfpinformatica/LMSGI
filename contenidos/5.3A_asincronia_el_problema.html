5.3A. Programación Asíncrona — El Problema ​

Objetivo: Entender visualmente por qué JavaScript necesita asincronía.
API utilizada: Ninguna — el problema se demuestra sin red.

¿Qué vamos a hacer? ​

Dos botones que hacen exactamente lo mismo: simular una operación lenta (3 segundos) y mostrar un resultado. La diferencia es cómo lo hacen:

  • El botón rojo lo hace de forma síncrona (bloqueante)
  • El botón verde lo hace de forma asíncrona

Hay un contador en pantalla que suma cada segundo. Sirve como indicador visual: si el contador se detiene, la página se ha congelado.

Nota técnica: No es posible bloquear JavaScript de verdad desde el navegador con una petición de red, porque el propio navegador lo impide. Para demostrar el bloqueo usamos un bucle que consume CPU durante 3 segundos, que tiene el mismo efecto práctico.

        <strong>¿Qué ha pasado?</strong><br><br>
        Con el código <strong>síncrono</strong>, JavaScript ocupó el hilo principal durante 3 segundos.
        Durante ese tiempo la página dejó de responder: el contador se congeló, no podías hacer clic
        en nada y el navegador posiblemente mostró un aviso de "página no responde".<br><br>
        Con el código <strong>asíncrono</strong>, JavaScript inició la espera y siguió ejecutando
        el resto del programa. El contador no se interrumpió en ningún momento.
        Cuando pasaron los 3 segundos, se ejecutó el código pendiente.<br><br>
        Las peticiones a APIs pueden tardar entre 100ms y varios segundos.
        Si bloquearan el hilo principal, ninguna web sería usable.
        Por eso <strong>todas las peticiones de red en JavaScript son asíncronas</strong>.
    </div>

    <script>
        // --- Contador ---
        let segundos = 0;
        setInterval(() => {
            segundos++;
            document.getElementById('contador').textContent = segundos;
        }, 1000);


        // --- Experimento 1: código síncrono (bloqueante) ---
        function operacionSincrona() {
            document.getElementById('resultado-sincrono').textContent = 'Trabajando...';

            // Bucle que ocupa el hilo durante 3 segundos
            // Mientras este bucle corre, NADA más puede ejecutarse
            const fin = Date.now() + 3000;
            while (Date.now() < fin) {
                // bucle vacío — bloquea el hilo principal
            }

            document.getElementById('resultado-sincrono').textContent = '✅ Terminado (pero la página se congeló)';
        }


        // --- Experimento 2: código asíncrono ---
        async function operacionAsincrona() {
            document.getElementById('resultado-asincrono').textContent = 'Trabajando...';

            // setTimeout dentro de una Promise: espera 3 segundos
            // sin bloquear el hilo principal
            await new Promise(resolve => setTimeout(resolve, 3000));

            document.getElementById('resultado-asincrono').textContent = '✅ Terminado (el contador no se detuvo)';
        }
    </script>

</body>
</html>

Qué observar ​

  1. Antes de pulsar nada: el contador sube cada segundo sin parar.
  2. Pulsa el botón rojo y observa el contador — se congela durante 3 segundos. Intenta hacer clic en el botón verde mientras tanto. No responde.
  3. Pulsa el botón verde y observa el contador — sigue subiendo con normalidad. Puedes seguir interactuando con la página.

La conclusión que debe quedar clara ​

JavaScript no puede esperar sin hacer nada. Si lo forzamos a esperar, congela toda la página. La solución es decirle "avísame cuando termines" en lugar de "espera hasta que termines". Eso es exactamente lo que hacen las Promesas y async/await.

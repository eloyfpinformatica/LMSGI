# 5.5. Trabajo con Datos JSON

**API utilizada:** Rick and Morty API — `https://rickandmortyapi.com`  
**Autenticación:** Ninguna  
**Objetivo:** Dominar `map()`, `filter()`, `reduce()` y transformación de datos JSON con una API real.

---

## ¿Qué vamos a hacer?

Una aplicación que carga personajes de Rick and Morty y permite filtrarlos y analizarlos usando los métodos de arrays de JavaScript.

Demostraremos:
- **`filter()`** — Filtrar personajes por estado (vivo/muerto/desconocido)
- **`map()`** — Extraer solo ciertos campos de cada personaje
- **`reduce()`** — Calcular estadísticas (contar por especie, por género, etc.)
- **Transformación de estructuras** — Adaptar el JSON de la API a nuestras necesidades

## Conceptos clave demostrados

### 1. **Transformación de estructura con `map()`**

El JSON que devuelve la API tiene esta forma:
```json
{
  "id": 1,
  "name": "Rick Sanchez",
  "status": "Alive",
  "species": "Human",
  "origin": {
    "name": "Earth (C-137)",
    "url": "https://..."
  }
}
```

Lo simplificamos a esto:
```javascript
{
  id: 1,
  nombre: "Rick Sanchez",
  estado: "alive",
  especie: "Human",
  origen: "Earth (C-137)",
  esHumano: true,
  estaVivo: true
}
```

**El código:**
```javascript
const personajesSimplificados = personajesCrudos.map(p => ({
    id: p.id,
    nombre: p.name,
    estado: p.status.toLowerCase(),
    especie: p.species,
    origen: p.origin.name,
    esHumano: p.species === 'Human'
}));
```

### 2. **Filtrado con `filter()`**

```javascript
// Solo personajes vivos
const vivos = personajes.filter(p => p.estado === 'alive');

// Solo humanos
const humanos = personajes.filter(p => p.especie === 'Human');

// Combinación de filtros
const humanosVivos = personajes
    .filter(p => p.especie === 'Human')
    .filter(p => p.estado === 'alive');
```

### 3. **Estadísticas con `reduce()`**

**Contar por categoría:**
```javascript
const porEstado = personajes.reduce((acum, p) => {
    acum[p.estado] = (acum[p.estado] || 0) + 1;
    return acum;
}, {});

// Resultado: { alive: 15, dead: 8, unknown: 2 }
```

**Coleccionar valores únicos:**
```javascript
const especies = personajes.reduce((acum, p) => {
    if (!acum.includes(p.especie)) acum.push(p.especie);
    return acum;
}, []);

// Resultado: ["Human", "Alien", "Robot", ...]
```

### 4. **Encadenamiento de métodos**

```javascript
const nombresDeMuertosHumanos = personajes
    .filter(p => p.estado === 'dead')      // 1. Solo muertos
    .filter(p => p.esHumano)               // 2. Solo humanos
    .map(p => p.nombre)                     // 3. Solo nombres
    .sort();                                // 4. Ordenar alfabéticamente

// Resultado: ["Abradolf Lincler", "Beth Smith", ...]
```

## Comparativa: forma imperativa vs funcional

**❌ Forma imperativa (con bucles):**
```javascript
const vivos = [];
for (let i = 0; i < personajes.length; i++) {
    if (personajes[i].estado === 'alive') {
        vivos.push(personajes[i]);
    }
}
```

**✅ Forma funcional (con filter):**
```javascript
const vivos = personajes.filter(p => p.estado === 'alive');
```

La segunda es más legible, menos propensa a errores y más fácil de encadenar con otras operaciones.
